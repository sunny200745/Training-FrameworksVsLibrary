Little about JS first

  • JavaScript® (often shortened to JS) is a lightweight, object-oriented language with first-class functions most known as the scripting language for Web pages.
  • The JavaScript standard is ECMAScript. As of 2012, all modern browsers fully support ECMAScript 5.1. Older browsers support at least ECMAScript 3.
  • ECMAScript is widely used for client-side scripting on the web, in the form of several well-known implementations such as JavaScript, JScript(Microsoft's version of JS) and ActionScript(Flash,AIR).
  • Today, "JavaScript" is a trademark of Oracle Corporation.

Why We Need Frameworks

  • Creating responsive, fluid, and maintainable interfaces for web apps isn’t as easy as one would imagine
  • When data to be sent back to the server,for result parsing, data stores to be updated, views to be re-rendered & so to keep everything synched with background.
  • The rise in the number of web apps has really made it apparent that we need better tools and frameworks.
  • Libraries : slot into your existing architecture and add specific functionality
  • Frameworks : give you an architecture (file structure, etc.) that you are meant to follow and, if you do, are intended to handle all common requirements

JMVC(aka )
  • JavaScriptMVC is an open-source framework containing the best ideas in jQuery development. It supports every browser that jQuery supports.
  • JMVC's goodies are broken down into four sub-projects
    • jQuery++ - A collection of useful DOM helpers and special events for jQuery
    • StealJS - JavaScript and CSS dependency management and build tools
    • FuncUnit - Functional and unit testing framework
    • DocumentJS - Documentation engine
StealJS
  • StealJS is a collection of command line and JavaScript client utilities that make building, packaging, and sharing JavaScript applications easy.
  • Dependency management (steal),steal loads modules into your app. Features:
    • Loads JS, CSS, Less, CoffeeScript, and client-side templates.
    • Parallel loading.
    • Works with scripts that don't use steal
  • Inculdes various like
    • steal.build - combines and minfies an application (or application's) resources into a small number of minified packages for faster downloading
    • steal.dev - logs messages cross browser. Messages are removed in production builds
    • steal.generate -makes building code generators extremely easy. Features
    • steal.clean - cleans your code and checks it against JSLint

JQuery++(Plus Plus)

  • jQuery++ is a collection of useful jQuery libraries that provide the missing functionality necessary to implement and organize large-scale jQuery applications.
  • DOM Helpers
      DOM helpers extend jQuery with extra functionality for manipulating the DOM. For example, [dimensions] lets you set the outer width and height of elements like:

    • Animate - Let jQuery use CSS animations.

  • Special Events like:

FuncUnit

  • FuncUnit tests web applications with a simple jQuery-like syntax. Via integration with Selenium and PhantomJS, you can run the same tests automated.
  • FuncUnit uses QUnit for organizing tests and assertions. But FuncUnit extends QUnit so you can:
    • Open a web page
    • Query for elements
    • Simulate a user action
    • Wait for a condition to be true
    • Get information about your page and run assertions
  • The FuncUnit Getting Started guide is a quick walkthrough of creating and running a test.

    • An extensible architecture

  • Backbone.js gives structure to web applications by providing models with key-value binding and custom events, collections with a rich API of enumerable functions, views with declarative event handling, and connects it all to your existing API over a RESTful JSON interface.
  • Backbone's only dependency is Underscore.js . For RESTful persistence, history support via Backbone.Router
    • Underscore.js
      • Underscore is a utility-belt library for JavaScript that provides a lot of the functional programming support that you would expect in Prototype.js (or Ruby)
    • Backbone.Router
      • Router Backbone.Router provides methods for routing client-side pages, and connecting them to actions and events

Introduction

  • When working on a web application that involves a lot of JavaScript, one of the first things you learn is to stop tying your data to the DOM.
  • For rich client-side applications, a more structured approach is often helpful
  • With Backbone, you represent your data as Models, which can be created, validated, destroyed, and saved to the server

Backbone

  • Events is a module that can be mixed in to any object, giving the object the ability to bind and trigger custom named events. Events do not have to be declared before they are bound, and may take passed arguments

Backbone.Model

  • Models are the heart of any JavaScript application, containing the interactive data as well as a large part of the logic surrounding it: conversions, validations, computed properties, and access control. You extend Backbone.Model with your domain-specific methods, and Model provides a basic set of functionality for managing changes

  • List of Model props can be viewed at Backbone Model
  • Collections are ordered sets of models. You can bind "change" events to be notified when any model in the collection has been modified, listen for "add" and "remove" events, fetch the collection from the server, and use a full suite of Underscore.js methods
  • Any event that is triggered on a model in a collection will also be triggered on the collection directly, for convenience. This allows you to listen for changes to specific attributes in any model in a collection, for example: documents.on("change:selected", ...)

  • List of Collections props/functions can be viewed at Backbone Collection

MVVM and View Models

  • Model-View-View Model (MVVM) is a design pattern for building user interfaces
  • It splits potentially sophisticated UI into three parts:
    • Model: Application’s data storage
    • View Model: A pure-code representation of the data and operations on a UI
    • View: Interactive UI representing the state of the view model

Introduction

  • AngularJS is a MVC framework that defines numerous concepts to properly organize your web application.
  • It enhances HTML by attaching directives to your pages with new attributes or tags and expressions in order to define very powerful templates directly in your HTML.
  • It also encapsulates the behavior of your application in controllers with the help of dependency injection.
  • Finally, utility code can easily be factorized into services that can be injected in your controllers
